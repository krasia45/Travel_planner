# 🗺️ AI 국내 복수 여행지 추천 및 자동 폴백(Fallback) 플래너
> **사용자가 입력한 특정 날짜를 기반으로 AI(Gemini) 모델이 국내 최적의 여행지를 큐레이션하고, 실시간 네이버 검색 API의 로컬 장소 데이터와 매시업(Mashup)하여 신뢰성 높은 여행 계획서를 자동 생성하는 백엔드 프로그램입니다.**

본 프로젝트는 AI 프롬프트 엔지니어링, 복수 데이터 파싱 아키텍처 구축, 외부 API 연동, 그리고 네트워크 장애에 대응하는 안정적인 **폴백(Fallback) 방어 프로그래밍**까지 백엔드 엔지니어가 거쳐야 할 필수적인 시스템 설계 과정을 집약하여 수행되었습니다.

---

## 🎯 1. 과제 목표 및 핵심 학습 성과

본 프로그램을 설계하고 고도화하며 달성한 핵심 기술적 목표는 다음과 같습니다.

* **REST API 및 HTTP 메서드의 이해**: 무상태성(Stateless) 통신 구조 하에서 데이터 생성을 위한 `POST` 요청(Gemini API)과 데이터 조회를 위한 `GET` 요청(Naver API)의 차이점을 명확히 이해하고 헤더 및 파라미터를 다르게 구성하여 구현했습니다.
* **LLM 출력 구조화(JSON Parsing)**: 비정형 텍스트를 반환하는 LLM의 특성을 통제하기 위해 프롬프트 엔지니어링을 적용하여 완전한 JSON 포맷 출력을 강제했습니다. 이를 통해 1차 데이터를 파싱하여 다음 파이프라인(네이버 장소 검색)의 입력값으로 사용하는 **데이터 연동 파이프라인**을 구축했습니다.
* **예외 처리 및 운영 안정성 확보**: 네트워크 단절, API 인증 실패(401/403), 쿼터 초과, 파싱 에러 발생 시 프로그램이 비정상 종료되지 않도록 `try-except` 예외 처리 및 재시도(Retry) 로직을 설계했습니다. 특정 API 실패 시에도 가용한 데이터를 바탕으로 최종 마크다운 리포트를 끝까지 생성합니다.
* **보안 내재화 (Environment Variables)**: 금융적 비용이 발생할 수 있는 소중한 API Secret Key의 유출을 방지하기 위해 코드 내 하드코딩을 원천 차단하고 `.env` 환경변수 관리 방식을 엄격히 적용했습니다.

## 🏗️ 3. 전체 시스템 아키텍처 및 데이터 흐름 (Data Flow)



본 프로그램은 단방향 흐름으로 동작하는 CLI 애플리케이션으로, 수집된 데이터를 최종적으로 정형화된 JSON 파일과 가독성 높은 마크다운(Markdown) 문서로 변환하여 저장합니다. 전체적인 프로세스 파이프라인은 다음과 같습니다.

1.  **사용자 입력 인터페이스 (Argument CLI):** 파이썬의 `argparse` 모듈을 통해 `YYYY-MM-DD` 포맷의 정격 날짜 인자를 수집 및 검증합니다.
2.  **AI 컨텍스트 분석 및 큐레이션 (LLM Processing):** 구글의 `gemini-2.5-flash` 모델을 활용하여 해당 날짜의 계절성, 기후 트렌드, 지역 축제 정보를 다각도로 분석한 후 복수의 최적 후보지를 JSON 구조체로 도출합니다.
3.  **실시간 로컬 데이터 매시업 (Data Mashup):** 분석된 후보 도시들을 식별자로 삼아 네이버 지역 검색 API를 동기식(Synchronous)으로 호출, 대중적인 실시간 맛집 리스트를 동적으로 결합합니다.
4.  **방어적 예외 제어 (Robust Fallback):** 네이버 API의 제한 초과나 데이터 공백 발생 시, 시스템 셧다운을 방지하고 서비스 연속성을 확보하기 위해 LLM의 도메인 지식 기반 백업 데이터로 즉시 전환(Fallback)합니다.
5.  **영속성 저장 및 최종 가공 (Persistence & Export):** 정형 데이터 보존을 위한 원본 JSON 스토리지 저장과 사용자 최종 인도용 마크다운 리포트 변환 작업을 병렬로 수행합니다.

## 🛠️ 4. 문제 해결 및 단계별 구현 과정 (Core Workflow)

과제를 성공적으로 완수하고 아키텍처를 고도화하기 위해 거친 핵심 개발 아젠다와 구현 과정입니다.

### [1단계] CLI 인터페이스 및 입력값 검증 (`argparse`)
* `argparse` 라이브러리를 사용해 `--date "YYYY-MM-DD"` 형태의 필수 옵션을 안전하게 처리하도록 구현했습니다.
* 정규표현식(`re`) 및 `datetime` 모듈을 연동하여, 사용자가 날짜 형식을 잘못 입력할 경우 직관적인 에러 가이드라인을 출력하고 안전하게 프로그램을 종료하도록 방어 코드를 작성했습니다.

### [2단계] `.env` 기반의 보안 강화 및 로드 트러블슈팅
* `.env` 파일을 생성하여 `GEMINI_API_KEY`, `NAVER_CLIENT_ID`, `NAVER_CLIENT_SECRET`을 격리했습니다.
* 개발 중 환경변수가 정상적으로 로드되지 않아 API 인증 에러(401)가 발생하는 현상을 겪었습니다. 이를 해결하기 위해 파일 경로를 절대 경로로 추적하거나 `dotenv.load_dotenv()`의 실행 시점을 프로그램 최상단으로 끌어올려 외부 유출 없는 안전한 인증 구조를 완성했습니다.

### [3단계] Gemini API 연동 및 JSON 구조화 처리
* 날짜 데이터를 프롬프트에 실어 Gemini 모델에 `POST` 요청을 보냈습니다.
* AI가 임의의 설명글을 붙여 JSON 파싱 에러를 유도하는 문제를 해결하기 위해, 프롬프트에 **"반드시 markdown 블록 없이 순수 JSON 스키마로만 응답하라"**는 제약 조건을 걸어 안전한 1차 JSON 객체를 확보했습니다.

### [4단계] 네이버 지역 검색 API 연동 및 데이터 토스
* 1차 JSON에서 파싱한 `recommended_city`를 키워드로 삼아 네이버 Local 검색 API에 `GET` 요청을 전송했습니다.
* 네이버 API 규격에 맞춰 `X-Naver-Client-Id`와 `X-Naver-Client-Secret`을 HTTP 헤더에 정확히 실어 나르도록 구현했으며, 응답받은 맛집 목록 중 상위 3곳의 필드를 정제해 바인딩했습니다.

### [5단계] 다원화된 에러 로깅 세션 추적 시스템 구현
* 프로그램 가동 중에 발생하는 경고나 외부 API 실패 현상을 실시간으로 추적하고, 최종 리포트에 고스란히 담아 개발자 및 사용자가 시스템 신뢰도를 모니터링할 수 있어야 했습니다.
* 글로벌 에러 추적 스택 배열(`ERROR_LIST`)을 선언하고, 단계별로 예외가 발생할 때마다 어떤 레이어(`Naver API`, `LLM Generation` 등)에서 어떤 유형의 문제가 생겼는지 정형화된 규격으로 기록하는 `record_error()` 유틸리티 함수를 빌드했습니다. 프로그램이 치명적인 이슈로 강제 종료되는 순간에도 `save_emergency_json()` 비상 저장 모듈이 가동되어 그때까지 쌓인 로그를 파일 시스템에 안착시킵니다.

## ✨ 5. 보너스 과제 최종 성과 요약

기본 과제 요건을 넘어, 실제 상용 서비스 수준의 안정성과 가독성을 갖추기 위해 아래 2가지 보너스 과제를 통합 설계했습니다.

### 1️⃣ 복수 지역 추천 (Multi-Destination Support) [보너스 1 완료]
* **기능 설명:** 기존의 단 한 곳만 추천하던 단조로운 방식에서 벗어나, AI가 입력된 날짜의 계절성과 지역 축제 트렌드를 분석한 뒤 **가장 최적의 후보 도시를 2~3곳 선정하여 리스트 배열 형태로 제공**합니다. 
* **구현 방식:** 프롬프트 내부 구조를 JSON Array 형태로 강제하고, 파이썬 코드 내에서 `for` 반복문(루프)을 가동하여 선정된 복수의 도시 정보와 맛집 데이터를 각각 독립적으로 매칭 및 누적 처리합니다.

### 2️⃣ 시스템 예외 대비 자동 폴백 시스템 (Robust Fallback Mechanism) [보너스 2 완료]
* **기능 설명:** 외부 API 환경은 트래픽 급증, 인증 만료, 순간적인 네트워크 단절 등으로 언제든 실패(Failure)할 위험이 존재합니다. 맛집 검색이 실패하더라도 시스템이 폭파되거나 리포트가 비어버리는 배드 유저 익스피리언스(Bad UX)를 원천 차단하고자 했습니다.
* **구현 방식:** 네이버 API 조회 결과가 비어있음을 감지하는 즉시 **`Fallback Triggered` 예외 세션 로그**를 작동시킵니다. 그리고 **AI(Gemini)가 프롬프트 생성 단계에서 미리 확보해 둔 '도시별 도메인 기반 자체 백업 맛집/특산물 정보(`fallback_restaurants`)'를 메모리 상에서 즉시 대체 결합(Binding)하도록 구현하여 무중단 환경을 완성했습니다.

- 한국 본토보다 온화한 기온을 유지하지만, 바람이 강할 수 있어 체감 온도는 쌀쌀할 수 있습니다. 맑은 날씨가 많습니다.

### 🥳 주요 행사/축제
- 감귤 따기 체험 (농장별)
- 동백꽃 개화 (일부 지역 시작)
- 한라산 단풍 절정기 이후 겨울 풍경 감상

### 💡 추천 이유
11월 중순 제주도는 늦가을의 정취와 초겨울의 이국적인 풍경을 동시에 경험할 수 있는 최적의 시기입니다. 황금빛 감귤이 가득한 과수원에서 감귤 따기 체험을 하거나, 서귀포 일대에서 피어나는 동백꽃을 미리 만나볼 수 있습니다. 본토보다 온화한 날씨 덕분에 야외 활동을 즐기기에도 부담이 적습니다.
