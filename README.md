# 🗺️ 국내 여행지 추천 및 맛집 연동 파이프라인 프로그램 (Travel Planner CLI)

본 프로젝트는 사용자가 입력한 특정 날짜의 계절성과 문화 행사 정보를 기반으로 **LLM(Google Gemini)을 활용해 최적의 국내 여행지를 추천**받고, 해당 지역의 **실시간 맛집 정보(Naver Local Search API)를 융합**하여 자동화된 여행 리포트를 생성하는 파이썬 기반 CLI 프로그램입니다.

---

## 🎯 1. 과제 목표 및 핵심 학습 성과

본 프로그램을 설계하고 트러블슈팅하며 달성한 핵심 기술적 목표는 다음과 같습니다.

*   **REST API 및 HTTP 메서드의 이해**: 무상태성(Stateless) 통신 구조 하에서 데이터 생성을 위한 `POST` 요청(Gemini API)과 데이터 조회를 위한 `GET` 요청(Naver API)의 차이점을 명확히 이해하고 헤더 및 파라미터를 다르게 구성하여 구현했습니다.
*   **LLM 출력 구조화(JSON Parsing)**: 비정형 텍스트를 반환하는 LLM의 특성을 통제하기 위해 프롬프트 엔지니어링을 적용하여 완전한 JSON 포맷 출력을 강제했습니다. 이를 통해 1차 데이터를 파싱하여 다음 파이프라인(네이버 장소 검색)의 입력값으로 사용하는 **데이터 연동 파이프라인**을 구축했습니다.
*   **예외 처리 및 운영 안정성 확보**: 네트워크 단절, API 인증 실패(401/403), 쿼터 초과, 파싱 에러 발생 시 프로그램이 비정상 종료되지 않도록 `try-except` 예외 처리 및 재시도(Retry) 로직을 설계했습니다. 특정 API 실패 시에도 가용한 데이터를 바탕으로 최종 마크다운 리포트를 끝까지 생성합니다.
*   **보안 내재화 (Environment Variables)**: 금융적 비용이 발생할 수 있는 소중한 API Secret Key의 유출을 방지하기 위해 코드 내 하드코딩을 원천 차단하고 `.env` 환경변수 관리 방식을 엄격히 적용했습니다.

*   **언어**: Python 3.10 이상
*   **인터페이스**: 터미널 기반 CLI (Command Line Interface)

## 🛠️ 3. 문제 해결 및 단계별 구현 과정

우리가 에러를 극복하며 파이프라인을 완성해 나간 실제 구현 여정입니다.

### [1단계] CLI 인터페이스 및 입력값 검증 (`argparse`)
*   `argparse` 라이브러리를 사용해 `--date "YYYY-MM-DD"` 형태의 필수 옵션을 안전하게 처리하도록 구현했습니다.
*   정규표현식(`re`) 및 `datetime` 모듈을 연동하여, 사용자가 날짜 형식을 잘못 입력할 경우 직관적인 에러 가이드라인을 출력하고 안전하게 프로그램을 종료하도록 방어 코드를 작성했습니다.

### [2단계] `.env` 기반의 보안 강화 및 로드 트러블슈팅
*   `.env` 파일을 생성하여 `GEMINI_API_KEY`, `NAVER_CLIENT_ID`, `NAVER_CLIENT_SECRET`을 격리했습니다.
*   개발 중 환경변수가 정상적으로 로드되지 않아 API 인증 에러(401)가 발생하는 현상을 겪었습니다. 이를 해결하기 위해 파일 경로를 절대 경로로 추적하거나 `dotenv.load_dotenv()`의 실행 시점을 프로그램 최상단으로 끌어올려 외부 유출 없는 안전한 인증 구조를 완성했습니다.

### [3단계] Gemini API 연동 및 JSON 구조화 처리
*   날짜 데이터를 프롬프트에 실어 Gemini 모델에 `POST` 요청을 보냈습니다.
*   AI가 임의의 설명글을 붙여 JSON 파싱 에러를 유도하는 문제를 해결하기 위해, 프롬프트에 **"반드시 markdown 블록 없이 순수 JSON 스키마로만 응답하라"**는 제약 조건을 걸어 안전한 1차 JSON 객체를 확보했습니다.

### [4단계] 네이버 지역 검색 API 연동 및 데이터 토스
*   1차 JSON에서 파싱한 `recommended_city`를 키워드로 삼아 네이버 Local 검색 API에 `GET` 요청을 전송했습니다.
*   네이버 API 규격에 맞춰 `X-Naver-Client-Id`와 `X-Naver-Client-Secret`을 HTTP 헤더에 정확히 실어 나르도록 구현했으며, 응답받은 맛집 목록 중 상위 5곳의 `name`, `address`, `category` 등의 필드를 정제해 바인딩했습니다.

### [5단계] 예외 처리 정책 및 무중단 리포트 발행
*   네이버 API 호출 실패나 검색 결과가 0건일 때 전체 프로세스가 멈추지 않도록 설계했습니다.
*   검색 실패 시 에러 로그를 `errors` 배열에 기록하고, 맛집 섹션은 `"데이터 없음"`으로 우아하게 대체 처리(Graceful Degradation)하여 최종 목적지인 마크다운 파일 생성을 끝까지 완수하도록 조치했습니다.
